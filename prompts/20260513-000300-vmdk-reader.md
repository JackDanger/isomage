## Human

Implement a VMDK (VMware Virtual Machine Disk) reader for isomage. Follow the
same conventions as vhd.rs. Public API: `detect<R: Read+Seek>` (restores
cursor) and `detect_and_parse<R: Read+Seek> -> Result<TreeNode, Error>`.
Add integration tests using qemu-img, update Cargo.toml and ci.yml.

## Assistant

### Key decisions

**Scope**: Only the sparse extent sub-type (magic `0x564d444b` at byte 0) is
implemented. Flat/raw extents have no magic and cannot be reliably detected
without external descriptor context. Descriptor-only files are plain text with
no binary magic. Both are rejected with `Error::BadMagic`.

**Endianness**: The SparseExtentHeader is all little-endian, unlike VHD which
is big-endian. `u32::from_le_bytes` / `u64::from_le_bytes` used throughout.

**Supported versions**: Only version 1 and version 3 are accepted per the
VMware VMDK specification; any other value returns `Error::UnsupportedVersion(v)`.

**Compressed images rejected**: `compress_algorithm == 1` (deflate /
streamOptimized) is detected in `read_header` and returns `Error::Compressed`
immediately. Decompressing grains would require a codec dep, which violates the
no-runtime-deps-in-default rule. This is checked before any tree is built so
the error is unambiguous.

**`file_location = None` always**: VMDK grain data is indexed through a grain
directory (GD) → grain table (GT) → grain indirection chain. The grains are not
contiguous, so we cannot express a single byte range for the virtual disk
contents. We report `file_location = None` and set `file_length = Some(capacity
* 512)` so consumers know the virtual size without claiming we can seek to the
data.

**File name `disk.vmdk`**: Used instead of `disk.img` to parallel the VHD
reader's `disk.img` convention while being format-specific. This helps
downstream consumers that might switch on the child name.

**TreeNode construction**: Used `TreeNode::new_file` followed by setting
`file_length` manually (same pattern as the dynamic VHD path in vhd.rs), rather
than `new_file_with_location`, because location is always None.

**Integration test scope**: Five round-trip tests cover detection, virtual size
reporting, file structure shape, version 1 acceptance, and twoGbMaxExtentSparse
detection. The twoGbMaxExtentSparse test uses qemu-img's info subcommand for
cross-validation (mirrors the VHD pattern). The tests skip cleanly when
`qemu-img` is absent and fail hard under `ISOMAGE_REQUIRE_TOOLS=1`.

**CI updates**: `vmdk` added to the individual feature build combo, the
all-features test/doc pass, the round-trip build and run steps (both apt and
pinned-docker jobs), and the MSRV feature loop. No new tools needed — qemu-img
was already installed for VHD tests.
