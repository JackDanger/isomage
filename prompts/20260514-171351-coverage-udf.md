### Key decisions

**Sector size must be 2048, not 512.** The module constant `SECTOR_SIZE = 2048`
is used for every seek — building synthetic test images with 512-byte sectors
would cause the parser to seek far past the end of the image and fail with an
I/O error. All test images use `const S: usize = 2048` to match the parser.

**Four tests added, all in a new `#[cfg(test)] mod tests` block at the end of
`src/udf.rs`:**

1. `parse_udf_synthetic_image_root_found` — happy path through the simple
   (non-metadata-partition) code. Uses `make_udf_image()` helper: VRS at
   sectors 16-18, AVDP at 256, PD+LVD+TD in sectors 257-259, FSD at 260,
   root FE (inline, parent-FID-only) at 261.

2. `parse_udf_no_avdp_after_vrs` — clobbers the AVDP tag to 0 and asserts
   the returned error mentions "AVDP not found".

3. `parse_udf_verbose_no_avdp` — same clobbered image but called with
   `verbose=true` to exercise the `eprintln!` at line 159 (the
   previously uncovered `if verbose` branch in the AVDP-not-found path).

4. `parse_udf_metadata_partition_path` — uses `make_udf_metadata_partition_image()`
   which adds a second physical partition (number 1, starts at sector 267)
   and places a type-2 partition map in the LVD identifying it as
   `*UDF Metadata Partition` with `meta_part_ref=1` and `meta_file_loc=0`.
   The metadata File Entry (tag 261, short AD) lives at sector 267 and
   points its first extent at sector 268. Sector 268 carries the FSD
   (tag 256) whose root ICB points to sector 269 (the root FE, inline).
   This drives execution through lines 294-338 — the entire metadata
   partition branch.
