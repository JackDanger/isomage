# PR #21 fix/pr9-review-feedback — Copilot review fixes

## Issues addressed

### CLI example in crate-level docs (medium)

Two places in `src/lib.rs` referenced `isomage -c huge.iso | head` despite the
crate having no CLI binary. Replaced both with library-focused phrasing that
doesn't imply a CLI exists.

### README consistency (low)

The "Parsers" section on line 258 said parsers "return an `io::Error` rather
than panic". The later "Stability" section correctly said "return `Err` (never
panic)". Changed line 258 to "return `Err`" to match.

### Semver note for `pub struct TreeNode` (medium)

README claimed "Adding new `pub` fields is non-breaking". This is incorrect for
a `pub struct` without `#[non_exhaustive]` — downstream code that constructs or
destructures `TreeNode` by value would break. Updated the README to say this is
a breaking change requiring a major version bump.

### `safe_join` symlink limitation (low)

`extract_node`'s safety doc overstated the protection: the lexical `starts_with`
check doesn't guard against pre-existing symlinks in the destination directory.
Added an explicit **Symlink limitation** paragraph recommending extraction into
a freshly created, empty directory to avoid this class of attack.

### Rebase conflict in `detect_and_parse_filesystem_verbose` doc (resolution)

The PR#9 fix tried to simplify the doc and revert the function to `&mut File`.
The parser-entry-generic change (now in main) had already made it generic.
Resolution: keep the generic signature, incorporate the cleaner doc wording from
the PR#9 fix. Result: "Useful for investigating images that fail to parse, or
when building your own diagnostic wrapper."
