# PR #15 VHD reader — redundant clone fix in test helper

## Issue addressed

### `parse_image` clones the image buffer twice (low)

`parse_image` in `tests/vhd_round_trip.rs` wrapped the bytes in
`Cursor::new(bytes.to_vec())` twice — once for `detect` and again for
`detect_and_parse`. For 10 MiB dynamic VHD tests this adds two unnecessary
20 MiB allocations per test.

Fixed by using `Cursor::new(bytes)` (borrowing the slice) for both calls,
resetting the cursor with `set_position(0)` between them. `Cursor<&[u8]>`
implements `Read + Seek`, so no API changes were needed.
