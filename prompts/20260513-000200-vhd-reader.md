## Human

Implement a VHD (Virtual Hard Disk) reader for the isomage Rust library.
Public API: `detect<R: Read+Seek>(r: &mut R) -> Result<(), Error>` and
`detect_and_parse<R: Read+Seek>(r: &mut R) -> Result<TreeNode, Error>`.
Follow patterns from `src/formats/ext.rs` and `src/formats/squashfs.rs`.

## Assistant

### Key decisions

**Detection via footer-at-end**: VHD detection seeks to `file_len - 512`
and checks for the `b"conectix"` cookie. This is the correct approach
because all three VHD subtypes (Fixed, Dynamic, Differencing) end with a
512-byte footer. The Dynamic subtype additionally has a copy at byte 0,
but reading from the end is the authoritative detection point.

**Fixed VHD: `file_location = Some(0)`**: A Fixed VHD places raw disk
data starting at byte 0, with the footer appended at the very end. This
means the entire virtual disk is one contiguous extent at offset 0 with
length `current_size`. This is analogous to how ISO 9660 files with a
single contiguous extent set `file_location`.

**Dynamic VHD: `file_location = None`**: Dynamic VHDs scatter data across
BAT-indexed blocks, each with a sector bitmap prefix. Reconstructing the
virtual disk requires walking the BAT and skipping per-block bitmaps —
non-trivial and not needed to satisfy the container-level tree. We report
`file_length = Some(current_size)` so callers know the virtual size, but
`file_location = None` because no single byte offset works.

**Differencing VHDs (disk_type 4) return `UnsupportedType(4)`**: Parent
resolution requires locating a parent VHD via locator entries (UTF-16 BE
paths with platform-specific formats), resolving relative paths, and
recursively reading the parent chain. This is deferred; a VHD that is
opened without its parent is useless to read anyway.

**Checksum: 1's complement of all bytes with checksum field zeroed**: The
VHD spec defines `checksum = ~(sum of all footer bytes treating bytes
64..68 as 0)`. We validate this on `detect_and_parse` to reject corrupt
images quickly. Detection only checks the 8-byte cookie (cheaper, enough
for the probe loop).

**TreeNode shape mirrors 7-Zip's VHD listing**: root `"/"` directory with
one child `"disk.img"`. This keeps the container level separate from the
inner filesystem — callers that want to read a filesystem inside the VHD
can invoke a second parser on the resolved extent.

**No BAT traversal in this PR**: The BAT walking code (for reconstructing
a Dynamic VHD's data as a contiguous byte stream) is not implemented.
Adding it would be the natural next step for enabling `cat_node` on
Dynamic VHD contents, but it belongs in a follow-up once we have a
`virtual_read` abstraction for fragmented extents.

**Round-trip tests use `qemu-img create -f vpc`**: `vpc` is qemu-img's
name for the VHD format (from "Virtual PC"). Fixed VHDs require `-o
subformat=fixed`; the default is Dynamic. Both subtypes are tested.
The round-trip tests skip gracefully when `qemu-img` is not installed
(normal on macOS without Homebrew) and fail hard under
`ISOMAGE_REQUIRE_TOOLS=1` (CI).
