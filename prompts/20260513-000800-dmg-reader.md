## Decisions

### koly trailer location
Detection seeks to `file_end − 512` and reads 4 bytes checking for `b"koly"`. This is the canonical UDIF detection idiom. `detect()` saves and restores the stream position.

### koly field offsets (corrected from spec)
The task spec listed `xml_offset` at koly offset 212 and `xml_length` at 220. Verified against a real `hdiutil`-produced DMG: the `data_fork_digest` field occupies bytes 84–212 (128 bytes), then 4 bytes reserved (212–216), then `xml_offset` at 216 and `xml_length` at 224. `sector_count` at 492 matched the spec.

### XML plist parsing without a dep
A minimal linear scanner finds `<key>blkx</key>`, the `<array>`, then each `<dict>`. Within each dict, `CFName` is preferred over `Name`. No XML dep needed; the Apple plist schema is regular enough for a substring scan.

### OOM guard
Added a 32 MiB sanity cap on `xml_length` before allocating, plus a bounds check that `xml_offset + xml_length ≤ file_length`. A corrupt koly with bogus `xml_length` would otherwise trigger a multi-GB allocation.

### No Mish/blkx binary decode
The `<data>` base64 blobs inside each blkx dict encode binary Mish extent tables describing how sectors are laid out. Decoding them requires understanding the Mish format and is not needed to list partition names. We don't decode them, keeping the reader zero-dep.

### Fallback tree when XML has no blkx
If the plist is empty or contains no `blkx` key (e.g. raw koly-only images used in tests), a single synthetic child `"disk.dmg"` is returned with `file_length = sector_count × 512`.

### hdiutil invocation
`hdiutil create -format UDRO` requires `-srcfolder`; it is not valid for blank image creation. The round-trip helper uses `-srcfolder $SRC_DIR` with a staged dummy file to produce a real UDRO DMG. Tests skip on Linux where `hdiutil` is absent. Hand-crafted koly buffers cover the no-hdiutil path.

### Version check location
`detect()` checks only the 4-byte magic (fast path). `detect_and_parse()` calls `read_koly()` which also validates `version == 4`, returning `Error::BadVersion(v)` otherwise.
